# 📊 E-commerce Funnel Analysis & A/B Testing

## 📌 Project Overview

In this project, I analyzed user behavior for an e-commerce platform to understand how users move through the purchase journey — from viewing products to adding them to cart and finally making a purchase.

Instead of just looking at numbers, I focused on identifying **where users drop off, why it happens, and how it can be improved**. The analysis was done using Python, and the results were visualized through an interactive Power BI dashboard.

---

## 🎯 Problem Statement

Businesses often struggle to understand:

* Where are users dropping off in the buying process?
* Which product categories perform better or worse?
* Does changing the experience (A/B testing) improve conversions?

This project answers these questions using real-world data.

---

## 🛠️ Tools Used

* **Python** (Pandas, basic analysis)
* **Power BI** (Dashboard creation)
* **Jupyter Notebook**

---

## 🔍 What I Did

* Cleaned and processed raw event-level data
* Built a **user-level funnel dataset**
* Analyzed conversion at each stage (View → Cart → Purchase)
* Performed **A/B testing** to compare two variants
* Identified category-level performance differences
* Created an interactive Power BI dashboard

---

## 📊 Key Insights

* A significant drop (~90%) occurs between **View → Cart stage**, indicating low engagement or weak product appeal
* **Product category plays a major role** in conversion — electronics perform better than others
* Some categories show very low conversion, suggesting issues with pricing, trust, or presentation
* A/B testing showed **no statistically significant difference** between variants

---

## 📈 Dashboard Features

* KPI cards showing user counts across funnel stages
* Funnel visualization to identify drop-off points
* A/B testing comparison chart
* Top-performing product categories by conversion rate
* Interactive filters for better exploration
